# UtilityBillingApp (Приложение для фиксации и расчёта коммунальных услуг)
Десктопное приложение на C# Приложение для автоматизации учёта показаний счётчиков, расчёта платежей за коммунальные услуги и ведения истории начислений. Позволяет управлять услугами, тарифами, показаниями и платежами в едином интерфейсе.

## О проекте

### Почему был создан этот продукт?
Проект разработан в рамках производственной практики по специальности как практическое решение для управления коммунальными платежами в малых жилых комплексах, управляющих компаниях или частных домохозяйствах.

### Какую проблему он решает?
Приложение автоматизирует процесс учёта коммунальных ресурсов, устраняя необходимость ведения бумажных журналов и ручного расчёта сумм. Оно решает проблему ошибок при расчётах, потери данных о показаниях и отсутствия единой истории платежей.

### В чём его ключевые преимущества?
* Простота использования — интуитивно понятный интерфейс, минимум полей для ввода
* Автоматизация расчётов — сумма к оплате вычисляется автоматически после ввода показаний
* Контроль тарифов — поддержка историчности тарифов, автоматическое применение актуальной цены
* Предотвращение дублирования — защита от повторного ввода показаний за один период
* Полная история — хранение всех показаний и платежей с возможностью фильтрации
* Локальное хранение — все данные хранятся в файле SQLite на вашем компьютере, не требуется интернет

## Быстрая установка и запуск

Вы можете запустить приложение за несколько минут, используя готовые файлы:

**1. Установите необходимое ПО:**
* Установите .NET Framework 4.8 по [этой ссылке](https://dotnet.microsoft.com/download/dotnet-framework/net48)
* Установка не требует дополнительных настроек

**2. Запустите приложение:**
* Скачайте готовый исполняемый файл [UtilityBillingApp.exe](https://github.com/Lision0k/-PracticeInSpecialty/blob/main/UtilityBillingApp.exe)
* Поместите его в отдельную папку (например, C:\UtilityBilling)
* Запустите приложение двойным кликом по UtilityBillingApp.exe

Важно: База данных (utility_billing.db) создастся автоматически в той же папке, где находится исполняемый файл. Все ваши данные будут храниться локально.

**3. Начните работу:**

После первого запуска в базе данных уже будут предустановлены базовые услуги:
* Электричество (кВт·ч)
* Холодная вода (м³)
* Горячая вода (м³)
* Газ (м³)
* Отопление (Гкал)

Вам остаётся только добавить тарифы и начать вносить показания.

## Использование

### Начало работы:
1. Запустите UtilityBillingApp.exe
2. Вы попадёте на главный экран с боковым меню навигации
3. По умолчанию открывается вкладка «Услуги»

### Добавление тарифа:
1. Перейдите во вкладку «Тарифы»
2. Нажмите «Добавить тариф»
3. Выберите услугу из списка
4. Укажите цену за единицу (например, 5.32 для электричества)
5. Задайте период действия тарифа
6. Нажмите «Сохранить»

### Ввод показаний:
1. Перейдите во вкладку «Показания»
2. Выберите услугу
3. Введите текущее показание счётчика
4. Убедитесь, что дата соответствует периоду снятия
5. Нажмите «Сохранить показание»
6. Система автоматически рассчитает сумму и добавит запись в историю платежей

### Просмотр истории платежей:
1. Перейдите во вкладку «История платежей»
2. При необходимости задайте фильтр по услуге или периоду
3. Нажмите «Сбросить фильтры» для отображения всех записей
4. Внизу отобразится итоговая сумма за выбранный период

### Расчёт за произвольный период:
1. Перейдите во вкладку «Рассчитать»
2. Выберите расчётный период
3. Отметьте услуги, которые хотите включить в расчёт
4. Нажмите «Рассчитать»

## Разработка и тестирование
Проект был разработан с использованием WindowsForms для создания пользовательского интерфейса и организации кода. Доступ к данным реализован через отдельный класс DatabaseHelper.

### Тестирование
В проекте проведено комплексное тестирование, включающее:
* Модульное тестирование: Проверка корректности работы отдельных компонентов, таких как расчёт баланса, валидация данных, работа фильтров.
* Интеграционное тестирование: Проверка взаимодействия между модулями (например, добавление транзакции → обновление баланса и бюджета). Тестирование проводилось на основе управляющего графа программы, охватывая основные пользовательские сценарии
